# Agent run measurements

`runs.jsonl` is an append-only evidence log generated by
`npm run agent:scorecard`. One row represents one substantial task, incident, or
handoff-resumption attempt. Do not use it as a vanity leaderboard.

`phase-runs.jsonl` is the phase-level companion generated by
`npm run agent:phase:record`. One row represents a product-build phase and
scores product lift, evidence quality, safety, token efficiency, and drift
control. A phase should not continue fan-out when drift control is below 70 or
mandatory gates are incomplete.

Compare agents/models only on similarly scoped tasks and risk tiers. A cheap
model that asks one more question but ships zero regressions may outperform an
expensive model that is faster and unsafe.

Never record prompts, secrets, customer PII, raw tokens, or private model chain
of thought. Record observable inputs, outputs, gates, interventions, duration,
cost, and defects.
